#### 阻塞io模型

用户线程在调用读写时阻塞，需要等待内核将数据拷贝至用户空间。

#### 非阻塞io模型

用户线程在调用读写时不阻塞，能返回，但是不一定有数据，需要用户线程不停轮询询问内核是否有数据。


#### 多路复用io模型

使用多路复用器来管理多个连接，用户线程只需要询问内核哪些连接有数据了。由内核去轮询连接状态。

#### 信号驱动io模型

用户线程发起一个io请求，会像socket注册一个信号函数，然后用户线程继续执行，当内核数据就绪时就会发送一个信号给用户线程，用户线程接受到信号之后，便在信号函数中调用io读写操作。

#### 异步io模型

用户线程只需要先发起一个请求，当接收内核返回的成功信号时表示 IO 操作已经完成，可以直接去使用数据了
